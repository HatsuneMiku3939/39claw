# References

This directory contains external reference materials used to shape the design and implementation of 39claw.

These references are not part of the 39claw application itself.
They exist to support architecture exploration, implementation decisions, and API integration work.

## Available References

### `codex-sdk`

Path: `docs/references/codex-sdk`

This reference contains the Codex SDK source, samples, and tests.
It is the main reference for Codex-native thread handling, request flow, and SDK integration patterns.

Start here:

- `docs/references/codex-sdk/README.md`

### `py-pimono`

Path: `docs/references/py-pimono`

This reference contains a Python local coding agent implementation inspired by `pi-mono`.
It is useful for studying application boundaries, orchestration structure, and agent-oriented architecture patterns.

Start here:

- `docs/references/py-pimono/README.md`
- `docs/references/py-pimono/ARCHITECTURE.md`

## Notes

- `codex-sdk` is the primary reference for Codex integration details.
- `py-pimono` is the primary reference for architecture ideas and boundary design.
- Local virtual environments or generated files inside reference directories should not be treated as first-class project artifacts.
